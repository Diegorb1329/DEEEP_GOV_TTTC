


An Evolutionary Lens on Web3/Open Source Software 🔎
https://gov.gitcoin.co/t/an-evolutionary-lens-on-web3-open-source-software/10293
    One of the coolest things about open source projects is how they evolve.  

Open source software has a deep evolutionary history. The open source ecosystem has evolved massively over the last 30 years.  This includes both:
1. Macro-evolution - OSS ecosystem (and all of it's subniches) maturing together
2. Micro-evolution - each individual project maturing (or dying)

If you study the source history of many of these projects, can you even see many compelling visuals of how code patches flow between branches. This is the beating heart of our digital infrastructure, being built before our eyes!

![git-model@2x|377x500, 75%](upload://mPWNQY9fWSCy8bwJeOvnFK7CkBC.png)

All web3 projects are open source.  What can we learn from the rich history of Open Source projects to teach us about what web3 can learn from how they've evolve?  How can we be inspired to write protocols that will survive & thrive, instead of protocols that are an evolution dead end?

In open source, we've seen that just having decent documentation, open source code, + offering forkability, a project (or an ecosystem) can evolve in many interesting & meaningful directions over time.

Take linux for example:

![44218-linuxdistrotimeline-7.2|690x497](upload://A8QbYLJOuHtjsubaZqbbmFT2T69.webp)

Over time, Linux has evolved to occupy many different niches.  Linux is used on
1. desktops
2. cloud servers
3. mobile phones
4. embedded devices.

[Here](https://hostingtribunal.com/blog/linux-statistics/#:~:text=In%202021%2C%20the%20lines%20of,million%20servers%20run%20on%20Linux.) are some stats on Linux' proliferation:

- In 2021, the lines of code submitted to the Linux Git repository reached 27.8 million.
- In 2021, 100% of the world’s top 500 supercomputers run on Linux.
- Out of the top 25 websites in the world, only 2 aren’t using Linux.
- 96.3% of the world’s top 1 million servers run on Linux.
- 90% of all cloud infrastructure operates on Linux and practically all the best cloud hosts use it.

Open Source Fork-ability has allowed evolution in Linux, and that has served Linux well!

# What Web3 can learn from Open Source?

I've been exploring an evolutionary perspective on web3, and I want to share it here.

I'm of the opinion that Linux was successful because it had

0. Utility
1. Interoperability
2. Open source
3. Forkability
4. Great documentation
5. A thriving developer community
1. Adoption network effects

How can web3 take what's great about Open Source, and add to it?

Lets take a look at how the web3 ecosystem has evolved over time visually and see what it's size/scale/scope is.

This is a photoshopped [tree of life](https://evogeneao.s3.amazonaws.com/images/tree_of_life/tree-of-life_2000.png) which has been modified to frame the birth & growth of web3 in an evolutionary perspective.

[![FPRg-F7UYAQKUjw|690x325](upload://knCUxuBkUAAuVc6kDTAoGGiOcOl.jpeg)](https://twitter.com/owocki/status/1509939046230872068)
*[( click here to view the discussion about this visual on twitter)](https://twitter.com/owocki/status/1509939046230872068)**

As the cliche goes: all models are wrong, but some are useful. 

I think this model is useful because it helps convey a few concepts:
1. the byzantine generals problem was the innovation trigger
2. An evolutionary perspective (from simplicity to more complexity & diversity):
    1. the bitcoin whitepaper created the first branch of open source money
    1. from there, layers of evolution happened on top of that in the form of BTC forks.
    1. once ethereum launched, a smart contract ecosystem began to evolve.
    1. upon that, an DEFI ecosystem & NFT ecosystem launched around the same time as an ecosystem of L2s & alt L1s
    2. further growth continues
3. evolution is not omnidirectional.  it expands in many different directions.
3. in nature, projects compete for scarce resources (energy) evolve according to natural selection.  in web3, projects compete for scarce resources (capital, attention) evolve according to memetic selection.
5. not every project (organism) makes it - over time, certain projects are evolutionary dead ends.  caught in bear markets (extinction events).
6. as groups of organisms (or projects) begin to co-habitate in the same evolutionary niche, they begin to co-evolve - either forming parasitic or symbiotic relationships.

One thing I think is neat about how web3 has grown is that it takes what made traditional open source work:

0. Utility
1. Interoperability
2. Open source
3. Forkability
4. Great documentation
5. A thriving developer community
1. Adoption network effects

& it adds a global coordination layer & a global incentivization layer on top of it.  This has led to some speculatular evolution over the years.

An Evolutionary Lens on Web3/Open Source Software 🔎
      

An Evolutionary Lens on Web3/Open Source Software 🔎
owocki

            <p>One of the coolest things about open source projects is how they evolve.</p>
<p>Open source software has a deep evolutionary history. The open source ecosystem has evolved massively over the last 30 years.  This includes both:</p>
<ol>
<li>Macro-evolution - OSS ecosystem (and all of it’s subniches) maturing together</li>
<li>Micro-evolution - each individual project maturing (or dying)</li>
</ol>
<p>If you study the source history of many of these projects, can you even see many compelling visuals of how code patches flow between branches. This is the beating heart of our digital infrastructure, being built before our eyes!</p>
<p></p><div class="lightbox-wrapper"><a class="lightbox" href="https://gov.gitcoin.co/uploads/db4391/original/2X/a/a00f2070303674cf373e0e65bdbf4fe5ce074aac.png" data-download-href="/uploads/short-url/mPWNQY9fWSCy8bwJeOvnFK7CkBC.png?dl=1" title="git-model@2x"><img src="/uploads/db4391/optimized/2X/a/a00f2070303674cf373e0e65bdbf4fe5ce074aac_2_282x375.png" alt="git-model@2x" data-base62-sha1="mPWNQY9fWSCy8bwJeOvnFK7CkBC" width="282" height="375" srcset="/uploads/db4391/optimized/2X/a/a00f2070303674cf373e0e65bdbf4fe5ce074aac_2_282x375.png, /uploads/db4391/optimized/2X/a/a00f2070303674cf373e0e65bdbf4fe5ce074aac_2_423x562.png 1.5x, /uploads/db4391/optimized/2X/a/a00f2070303674cf373e0e65bdbf4fe5ce074aac_2_564x750.png 2x" data-small-upload="/uploads/db4391/optimized/2X/a/a00f2070303674cf373e0e65bdbf4fe5ce074aac_2_10x10.png"></a></div><p></p>
<p>All web3 projects are open source.  What can we learn from the rich history of Open Source projects to teach us about what web3 can learn from how they’ve evolve?  How can we be inspired to write protocols that will survive &amp; thrive, instead of protocols that are an evolution dead end?</p>
<p>In open source, we’ve seen that just having decent documentation, open source code, + offering forkability, a project (or an ecosystem) can evolve in many interesting &amp; meaningful directions over time.</p>
<p>Take linux for example:</p>
<p></p><div class="lightbox-wrapper"><a class="lightbox" href="https://gov.gitcoin.co/uploads/db4391/original/2X/f/fd4e21557ffee46f5bfe180950c6c9410fd49699.webp" data-download-href="/uploads/short-url/A8QbYLJOuHtjsubaZqbbmFT2T69.webp?dl=1" title="44218-linuxdistrotimeline-7.2"><img src="/uploads/db4391/optimized/2X/f/fd4e21557ffee46f5bfe180950c6c9410fd49699_2_690x497.webp" alt="44218-linuxdistrotimeline-7.2" data-base62-sha1="A8QbYLJOuHtjsubaZqbbmFT2T69" width="690" height="497" srcset="/uploads/db4391/optimized/2X/f/fd4e21557ffee46f5bfe180950c6c9410fd49699_2_690x497.webp, /uploads/db4391/optimized/2X/f/fd4e21557ffee46f5bfe180950c6c9410fd49699_2_1035x745.webp 1.5x, /uploads/db4391/optimized/2X/f/fd4e21557ffee46f5bfe180950c6c9410fd49699_2_1380x994.webp 2x" data-small-upload="/uploads/db4391/optimized/2X/f/fd4e21557ffee46f5bfe180950c6c9410fd49699_2_10x10.png"></a></div><p></p>
<p>Over time, Linux has evolved to occupy many different niches.  Linux is used on</p>
<ol>
<li>desktops</li>
<li>cloud servers</li>
<li>mobile phones</li>
<li>embedded devices.</li>
</ol>
<p><a href="https://hostingtribunal.com/blog/linux-statistics/#:~:text=In%202021%2C%20the%20lines%20of,million%20servers%20run%20on%20Linux.">Here</a> are some stats on Linux’ proliferation:</p>
<ul>
<li>In 2021, the lines of code submitted to the Linux Git repository reached 27.8 million.</li>
<li>In 2021, 100% of the world’s top 500 supercomputers run on Linux.</li>
<li>Out of the top 25 websites in the world, only 2 aren’t using Linux.</li>
<li>96.3% of the world’s top 1 million servers run on Linux.</li>
<li>90% of all cloud infrastructure operates on Linux and practically all the best cloud hosts use it.</li>
</ul>
<p>Open Source Fork-ability has allowed evolution in Linux, and that has served Linux well!</p>
<h1>
<a name="what-web3-can-learn-from-open-source-1" class="anchor" href="https://gov.gitcoin.co#what-web3-can-learn-from-open-source-1"></a>What Web3 can learn from Open Source?</h1>
<p>I’ve been exploring an evolutionary perspective on web3, and I want to share it here.</p>
<p>I’m of the opinion that Linux was successful because it had</p>
<ol start="0">
<li>Utility</li>
<li>Interoperability</li>
<li>Open source</li>
<li>Forkability</li>
<li>Great documentation</li>
<li>A thriving developer community</li>
<li>Adoption network effects</li>
</ol>
<p>How can web3 take what’s great about Open Source, and add to it?</p>
<p>Lets take a look at how the web3 ecosystem has evolved over time visually and see what it’s size/scale/scope is.</p>
<p>This is a photoshopped <a href="https://evogeneao.s3.amazonaws.com/images/tree_of_life/tree-of-life_2000.png">tree of life</a> which has been modified to frame the birth &amp; growth of web3 in an evolutionary perspective.</p>
<p><a href="https://twitter.com/owocki/status/1509939046230872068"><img src="/uploads/db4391/original/2X/8/8ed738d8df75d77a83153cae958e9f73bb057931.jpeg" alt="FPRg-F7UYAQKUjw" data-base62-sha1="knCUxuBkUAAuVc6kDTAoGGiOcOl" width="690" height="325"></a><br>
<em><a href="https://twitter.com/owocki/status/1509939046230872068">( click here to view the discussion about this visual on twitter)</a></em>*</p>
<p>As the cliche goes: all models are wrong, but some are useful.</p>
<p>I think this model is useful because it helps convey a few concepts:</p>
<ol>
<li>the byzantine generals problem was the innovation trigger</li>
<li>An evolutionary perspective (from simplicity to more complexity &amp; diversity):
<ol>
<li>the bitcoin whitepaper created the first branch of open source money</li>
<li>from there, layers of evolution happened on top of that in the form of BTC forks.</li>
<li>once ethereum launched, a smart contract ecosystem began to evolve.</li>
<li>upon that, an DEFI ecosystem &amp; NFT ecosystem launched around the same time as an ecosystem of L2s &amp; alt L1s</li>
<li>further growth continues</li>
</ol>
</li>
<li>evolution is not omnidirectional.  it expands in many different directions.</li>
<li>in nature, projects compete for scarce resources (energy) evolve according to natural selection.  in web3, projects compete for scarce resources (capital, attention) evolve according to memetic selection.</li>
<li>not every project (organism) makes it - over time, certain projects are evolutionary dead ends.  caught in bear markets (extinction events).</li>
<li>as groups of organisms (or projects) begin to co-habitate in the same evolutionary niche, they begin to co-evolve - either forming parasitic or symbiotic relationships.</li>
</ol>
<p>One thing I think is neat about how web3 has grown is that it takes what made traditional open source work:</p>
<ol start="0">
<li>Utility</li>
<li>Interoperability</li>
<li>Open source</li>
<li>Forkability</li>
<li>Great documentation</li>
<li>A thriving developer community</li>
<li>Adoption network effects</li>
</ol>
<p>&amp; it adds a global coordination layer &amp; a global incentivization layer on top of it.  This has led to some speculatular evolution over the years.</p>
